# Exercise 1: SDLC Models and Testing Strategy

## Purpose

These exercises help you understand how different SDLC models affect testing strategy. The goal is to develop **intuition** for adapting your testing approach based on the development model.

---

## Exercise 1.1: Waterfall Model Testing

### Task

Imagine a project using Waterfall model:

**Phases**:
1. Requirements (2 weeks)
2. Design (3 weeks)
3. Implementation (8 weeks)
4. Testing (4 weeks)
5. Deployment (1 week)

1. **Plan Testing**:
   - When would you start test planning?
   - What would you do in each phase?
   - How would you prepare for testing phase?

2. **Testing Phase Activities**:
   - What would you test?
   - How would you organize testing?
   - What challenges would you face?

3. **Reflect on Waterfall**:
   - What are the advantages for testing?
   - What are the disadvantages?
   - How would you mitigate challenges?

### Reflection

- How does late testing affect defect discovery?
- What can you do in earlier phases to help testing?
- How do you ensure quality in Waterfall?

## Exercise 1.2: V-Model Testing

### Task

Map testing to V-Model phases:

**Left Side (Development)**:
- Requirements
- System Design
- Detailed Design
- Implementation

**Right Side (Testing)**:
- Acceptance Testing
- System Testing
- Integration Testing
- Unit Testing

1. **Map Testing Activities**:
   - What testing happens at each level?
   - How does it correspond to development?
   - What test documents are created when?

2. **Create Test Plan**:
   - Plan tests for each level
   - Show how they trace to requirements
   - Document entry/exit criteria

3. **Reflect on V-Model**:
   - How does parallel planning help?
   - What's the value of traceability?
   - What are the limitations?

### Reflection

- How does V-Model improve on Waterfall?
- Why is traceability important?
- How does this model help with verification?

## Exercise 1.3: Agile Model Testing

### Task

Imagine an Agile project with 2-week sprints:

**Sprint Structure**:
- Sprint Planning (Day 1)
- Development (Days 2-8)
- Testing (Days 9-10)
- Sprint Review (Day 10)
- Retrospective (Day 10)

1. **Plan Testing for Sprint**:
   - When do you test?
   - How do you test continuously?
   - What's your daily testing routine?

2. **Test Activities**:
   - What do you do during development?
   - What do you do during testing days?
   - How do you handle incomplete features?

3. **Continuous Testing**:
   - How do you test as features are built?
   - How do you test integration?
   - How do you handle changes?

### Reflection

- How is Agile testing different from Waterfall?
- What's the value of continuous testing?
- How do you maintain quality in fast iterations?
- What challenges does Agile present for testing?

## ðŸ’¡ Key Insights to Carry Forward

- **SDLC models determine testing strategy** - Different models, different approaches
- **Waterfall tests late** - Sequential, formal, challenges with late defect discovery
- **V-Model tests parallel** - Verification at each level, better traceability
- **Agile tests continuously** - Integrated, adaptive, fast feedback
- **Context matters** - Choose model (and testing approach) based on project
- **Hybrids are common** - Adapt testing to the actual process
- **Flexibility is key** - Be able to work in different models
